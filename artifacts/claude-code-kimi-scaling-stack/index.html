Scaling, reconsidered

Scaling didn’t stop —
it just grew more dimensions.

The Kimi team’s pitch is simple to state and harder to live with: bigger models and more tokens are no longer the only knobs. Behind a single loss curve sit five compounding levers — token efficiency, long context, agent swarms, multimodal pretraining, and a new kind of residual connection. This is an interactive read on what each lever actually does, and why a frontier open model has to pull on all of them at once.

02 — Lever one · Token efficiencyShift the loss curve to the left.

Token efficiency isn’t cost-cutting; it’s an intelligence multiplier. If you can squeeze 2× the learning out of the same data, your fixed pile of high-quality tokens effectively doubles — right when the field is hitting a data wall.

MuonClip = Muon + QK-Clip

A second-order optimizer that scales.

Muon is a second-order optimizer that transforms each gradient update so entries are roughly orthogonal — a stark departure from Adam. Implemented well, Kimi reports it roughly doubles token efficiency.

But at a trillion parameters Muon blew up: max attention logits exploded past 1,000 and loss diverged. QK-Clip rescales the query and key projections of each attention head using the observed max logit in the forward pass, holding it inside a sane range without changing the loss curve.

Why it matters
High-quality tokens are finite; better optimization compounds the ceiling.
What QK-Clip fixes
Numerical blow-up at scale — not the loss itself.
Drop-in?
Yes — positioned as a replacement for AdamW.

03 — Lever two · Long contextLoss should keep dropping deeper into the prompt.

A hidden gem from the Kaplan paper: transformers don’t just beat LSTMs in aggregate loss — they keep improving as the context grows. Kimi Linear pushes that further with a mix of linear and full attention plus a finer-grained forgetting mechanism.

Kimi Linear · Kimi Delta Attention

Per-channel decay instead of one global knob.

Vanilla linear attention shares a single scalar decay factor — you either forget almost everything or you remember almost everything. Kimi Delta Attention replaces that scalar with a diagonal matrix, letting each channel decay at its own rate.

Some channels persist for hundreds of thousands of tokens, while others refresh quickly to track new information. Mixed with full attention in a 1:3 ratio, Kimi Linear is reported to beat full attention across short and long context — while staying cheaper at scale.

Why mix?
Pure linear attention is fast but expressively weaker than full attention.
Math caveat
The chunk-wise reformulation is exact, not an approximation.
Why it matters
Agent runs that go for days need useful memory, not just long memory.

05 — Lever four · Multimodal pretrainingVision and text, fused from day one.

Open multimodal models typically bolt vision onto a finished text model. K2.5 instead trains vision and text jointly from 0% of pretraining — early fusion. The surprising part: when aligned in a shared representation, the two modalities help each other.

Early fusion · zero vision SFT

Two modalities, one representation space.

The conventional recipe is late fusion: train a strong text base on 20T tokens, then post-train on 2T tokens of vision data. K2.5 mixes vision and text tokens from step zero, so they share an embedding space.

Two reported asymmetries: vision-only training improves text benchmarks even though no math or code tokens were involved, and a strong text base pulls vision performance up without using any vision supervised fine-tuning data at all (“zero vision SFT”).

06 — Lever five · Attention residualsResidual connections, but rotated 90°.

Ilya’s framing: a standard residual connection is an LSTM rotated 90 degrees, running across the depth dimension. If we already replaced LSTMs with attention in the temporal dimension, why not also replace residuals with attention in the depth dimension?

Attention over previous layers

Each layer attends over all the layers below it.

A standard residual just adds the previous layer’s output to the current one — a fixed-weight recurrence over depth. Attention residuals let the current layer attend over all previous hidden states, weighting them dynamically.

To keep compute and memory tractable, the team groups layers into blocks; attention residuals cross block boundaries while standard residuals operate inside each block.

Reported gain
+24% token efficiency over a strong baseline.
What it helps most
Coding, math, and reasoning-heavy benchmarks.
Released
As a tech report alongside the talk (per the speaker).
